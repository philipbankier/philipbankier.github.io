Core Beliefs & Positions

Canonical Type Authority
Action classification is a first-class architectural concern. Types must be canonical at runtime boundaries. Audit trails built into authority systems. (CONTRACT v1.9 / ADR 0016)
Production-Grade Reliability
Record/replay is fundamental for deterministic agent testing. Trial execution should be isolated. Developer overrides must be explicit and receipted — never silent.
Memory Architecture
Memory systems need restartability for production. Proofs must be source-grounded. Temporal scoping essential for access control. Batched writes necessary for scale.
Governed Learning
Skills should be governed, not free-form. Outcomes need reliability projections. Learning must be reversible. Evaluation must co-evolve with the system.
Observability
Runtime event observation deserves its own architectural seam. Execution and observation must be cleanly separated. Event sourcing enables downstream integration.
